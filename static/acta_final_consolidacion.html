Acta Final — Reglas de Consolidación por Campo

Lógica base: Para cada numero_conteo seleccionado, se verifica si existe algún registro en INV_INVENTARIO_FISICO_VS_SISTEMA con OBSERVACION_4 IS NOT NULL.
• Si NO existe → se usan los valores ya calculados de ACTA_PRELIMINAR_TBL.
• Si SÍ existe → los campos de inventario se recalculan desde los campos _4 de INV_INVENTARIO_FISICO_VS_SISTEMA.
Los campos de personal y SAP siempre vienen de ACTA_PRELIMINAR_TBL independientemente de OBSERVACION_4.

Variables usadas en las queries:
• :numeros_conteo = lista de numero_conteo seleccionados por el usuario
• :piqueo_ids = lista de piqueo_id correspondientes a los conteos seleccionados
• :conteos_sin_obs4 = conteos sin cambios en tercer conteo (OBSERVACION_4 IS NULL para todos sus items)
• :conteos_con_obs4 = conteos con al menos un item con OBSERVACION_4 IS NOT NULL

Query 0 — Detección de conteos con OBSERVACION_4

-- Detecta qué conteos tienen items modificados en el tercer conteo SELECT DISTINCT numero_conteo FROM inv_inventario_fisico_vs_sistema WHERE numero_conteo IN :numeros_conteo AND observacion_4 IS NOT NULL
Los numero_conteo que aparezcan aquí se procesan con la rama recalcular desde _4. Los que NO aparezcan usan directamente ACTA_PRELIMINAR_TBL.

Query del Listado Principal (acta_final view)

-- Muestra solo conteos en TERCER_CONTEO que tienen acta preliminar generada SELECT p.piqueo_id, p.numero_conteo, p.estado, p.fecha_inicio, p.fecha_fin, nombre_empleado_func(p.usuario_responsable) AS nm_responsable, p.usuario_creacion, p.centro, p.almacen FROM INV_PIQUEOS_INVENTARIO_TBL p WHERE UPPER(p.estado) = 'TERCER_CONTEO' AND EXISTS ( SELECT 1 FROM acta_preliminar_tbl ap WHERE ap.numero_conteo = p.numero_conteo ) ORDER BY p.fecha_inicio DESC
